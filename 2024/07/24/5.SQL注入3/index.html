    • 1.宽字节注入

    1.1什么是宽字节注入

    宽字节是相对于ascII这样单字节而言的；像 GB2312、GBK、GB18030、BIG5、Shift_JIS 等这些都是常说的宽字节，实际上只有两字节。GBK 是一种多字符的编码，通常来说，一个 gbk 编码汉字，占用2个字节。一个 utf-8 编码的汉字，占用3个字节。

    转义函数：为了过滤用户输入的一些数据，对特殊的字符加上反斜杠“\”进行转义；

    宽字节注入指的是 mysql 数据库在使用宽字节（GBK）编码时，会认为两个字符是一个汉字（前一个ascii码要大于128（比如%df），才到汉字的范围），而且当我们输入单引号时，mysql会调用转义函数，将单引号变为’，其中\的十六进制是%5c,mysql的GBK编码，会认为%df%5c是一个宽字节，也就是’運’，从而使单引号闭合（逃逸），进行注入攻击。

    1.3实战:sqli-labs第32关

    1. 无论我们在id=1后面加任意字符都不会显示报错，说明字符可能被转义了
    2. 查看源代码，发现了gbk编码，说明存在宽字节注入
    3. ?id=1%df%27在单引号前面加上%df,结果发生报错
    4. 在后面加上–+注释符并使用order by判断字段数
    5. 字段数为3，使用union联合查询查看当前数据库

    2.二次注入

    2.1什么是二次注入

    二次注入是指已存储（数据库、文件）的用户输入被读取后再次进入到 SQL 查询语句中导致的注入。
    二次注入是sql注入的一种，但是比普通sql注入利用更加困难，利用门槛更高。普通注入数据直接进入到 SQL 查询中，而二次注入则是输入数据经处理后存储，取出后，再次进入到 SQL 查询。

    2.2二次注入的原理

    2.3实战：sqli-labs第24关

    1. 输入admin’#和123456，显示登录失败
    2. 点击创建新用户
    3. 用户名:admin’#,密码:123
    4. 在数据库查看admin和admin’#的密码
    5. 使用admin’#和密码123登录并将密码修改成123456
    6. 查看数据库，此时admin原来的密码变成了123456
    7. Username直接从数据库中取出，没有经过转义处理。在更新用户密码的时候其实执行了下面的命令：

    “UPDATE users SET PASSWORD=‘123456’ where username=‘admin’#’ and password=’$curr_pass’”。
    因为我们将问题数据存储到了数据库，而程序再取数据库中的数据的时候没有进行二次判断便直接带入到代码中（#号将后面的单引号注释掉了），从而造成了二次注入。
